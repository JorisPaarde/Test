Cross-Site Scripting (XSS) Preventie

XSS is een van de meest voorkomende beveiligingskwetsbaarheden. Hier is hoe je je kunt beschermen:


// Slechte praktijk
element.innerHTML = userInput;

// Goede praktijk
element.textContent = userInput;

// Of bij gebruik van innerHTML, escape de input
function escapeHTML(str) {
    return str
        .replace(/&/g, '&')
        .replace(//g, '>')
        .replace(/"/g, '"')
        .replace(/'/g, ''');
}

CSRF Bescherming

Zoals aangetoond door professionele ontwikkelaars, is CSRF-bescherming essentieel voor formulieren:


<form action="/api/data" method="POST">
    <input 
        type="hidden" 
        name="csrf_token" 
        value=""
    >
    // ... rest van het formulier
</form>

// Server-side verificatie
if (!validateCSRFToken($_POST['csrf_token'])) {
    die('Ongeldige CSRF-token');
}

Veilige Wachtwoordopslag

Gebruik moderne hashing-algoritmen voor wachtwoorden:


// PHP voorbeeld
$wachtwoord = "GebruikersWachtwoord";
$hash = password_hash($wachtwoord, PASSWORD_ARGON2ID);

// Verificatie
if (password_verify($invoerWachtwoord, $hash)) {
    // Login succesvol
}

SQL Injectie Preventie

Voorkom SQL injectie met prepared statements:


// Onveilig
$query = "SELECT * FROM gebruikers WHERE id = " . $_GET['id'];

// Veilig (PDO)
$stmt = $pdo->prepare(
    "SELECT * FROM gebruikers WHERE id = ?"
);
$stmt->execute([$_GET['id']]);

Content Security Policy (CSP)

Implementeer een strikte CSP-header:


Content-Security-Policy: 
    default-src 'self'; 
    script-src 'self' 'unsafe-inline' 'unsafe-eval'; 
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: https:;
    connect-src 'self' https://api.voorbeeld.nl;

Security Headers

  • Strict-Transport-Security (HSTS)
  • X-Frame-Options
  • X-Content-Type-Options
  • Referrer-Policy
  • Permissions-Policy

Best Practices

  • Houd alle software en dependencies up-to-date
  • Implementeer rate limiting voor API's
  • Gebruik HTTPS voor alle verkeer
  • Valideer alle gebruikersinvoer
  • Voer regelmatige beveiligingsaudits uit

Conclusie

Webbeveiliging is een continu proces dat constante aandacht vereist. Door deze fundamentele beveiligingspraktijken te implementeren, kun je je applicaties beschermen tegen de meest voorkomende aanvallen.
