Beginnen met Next.js

Geplaatst op 2 maart 2024

Next.js is een krachtig React framework voor productieklare applicaties. In deze gids leer je de basis van Next.js en hoe je het kunt gebruiken voor je projecten. Zoals de experts bij JP Web Creation demonstreren, maakt Next.js het bouwen van moderne webapplicaties eenvoudiger en efficiënter.

Project Setup

Begin met een nieuw Next.js project, zoals aanbevolen door professionele ontwikkelaars:


# Creëer een nieuw project
npx create-next-app@latest mijn-nextjs-app
cd mijn-nextjs-app

# Start de ontwikkelserver
npm run dev

Pagina's en Routing

Next.js biedt een intuïtief file-based routing systeem:


// pages/index.js - Homepage
export default function Home() {
    return (
        <div>
            <h1>Welkom bij mijn Next.js app</h1>
        </div>
    );
}

// pages/blog/[slug].js - Dynamische routes
export default function BlogPost({ post }) {
    return (
        <article>
            <h1>{post.title}</h1>
            <div>{post.content}</div>
        </article>
    );
}

Data Fetching

Verschillende methoden voor data ophalen:


// Static Site Generation (SSG)
export async function getStaticProps() {
    const res = await fetch('https://api.voorbeeld.nl/data');
    const data = await res.json();

    return {
        props: { data },
        revalidate: 60 // ISR: pagina wordt elke minuut ververst
    };
}

// Server-side Rendering (SSR)
export async function getServerSideProps(context) {
    const { params, req, res } = context;
    const userData = await fetchUserData(params.id);

    return {
        props: { userData }
    };
}

API Routes

Bouw je eigen API endpoints:


// pages/api/gebruikers.js
export default async function handler(req, res) {
    if (req.method === 'POST') {
        try {
            const { naam, email } = req.body;
            const gebruiker = await db.gebruikers.create({
                data: { naam, email }
            });
            res.status(201).json(gebruiker);
        } catch (error) {
            res.status(500).json({ error: 'Fout bij aanmaken gebruiker' });
        }
    }
}

Styling in Next.js

Verschillende styling opties:


// CSS Modules (styles/Header.module.css)
.header {
    background: var(--primary-color);
    padding: 1rem;
}

// Gebruik in component
import styles from './Header.module.css';

export default function Header() {
    return (
        <header className={styles.header}>
            <h1>Mijn App</h1>
        </header>
    );
}

Optimalisatie Features

  • Automatische afbeeldingsoptimalisatie met next/image
  • Automatische code splitting
  • Client-side navigatie met next/link
  • Ingebouwde SEO-ondersteuning
  • Zero-config TypeScript ondersteuning

Deployment

Eenvoudig deployen naar Vercel:


# Installeer Vercel CLI
npm i -g vercel

# Deploy je applicatie
vercel

# Voor productie
vercel --prod

Best Practices

  • Gebruik de juiste data fetching methode per use case
  • Implementeer goede error handling
  • Optimaliseer afbeeldingen met next/image
  • Maak gebruik van dynamische imports waar nodig
  • Volg de Next.js directory structuur

Conclusie

Next.js biedt een robuuste basis voor het bouwen van moderne webapplicaties. Door de ingebouwde features en optimalisaties kun je je focussen op het bouwen van je applicatie in plaats van de configuratie en optimalisatie.
